内心独白一下：现在我到底会了什么？

你会写 composite pass 做全屏后处理（第 1 章），会用 `shaders.properties` 搭出配置界面（第 2 章和第 10 章），会在 gbuffers pass 里写地形着色和光照（第 2~4 章），会延迟渲染和 SSAO（第 5~6 章），会 shadow mapping 和 PCF 软阴影（第 7~8 章），会自定义天空、体积光、雾（第 9~10 章），会水面反射和 Fresnel（第 11 章），会 PBR 材质（第 12 章），会 Bloom、色调映射、抗锯齿（第 13~15 章），会体积云和视差贴图（第 16~17 章），会其他 gbuffers pass 和维度分离（第 18~19 章），还能读完 BSL 的架构（第 20 章）。

但是，教程的能力地图到这里就结束了。还有更多东西在前面等着你。

## 社区：你不是一个人在干活

图形编程有一个共同点：几乎每个人的第一个 shader 都是黑的。

然后你在社区里问了一句，有人回你："你忘了写 `outColor = ...`"。你加上去，屏幕变了颜色——那一刻你是世界上最开心的人。

> 社区是你 debug 的最快方式，也是你学习别人偷懒技巧的最好来源。

下面是你应该去的地方：

**ShaderLABS Discord** — 这是全球 Minecraft shader 开发者最集中的地方。BSL 作者 Capt Tatsu、Iris 核心贡献者、各路光影作者都在里面。不管你是用英文还是翻译软件，贴代码、问报错、看别人怎么解决的，这里都是你的主战场。

**Iris 官方 Discord** — Iris 加载器本身的讨论和 bug 报告。如果你怀疑你的 shader 代码没问题、是 Iris 的锅，来这里问最合适。

**MCBBS / 中文光影&资源包社区** — 对中文开发者来说，这里有许多人用中文讨论光影开发。你可以在这里找到测试者、翻译帮忙者和同行交流。

交流的时候注意一件事：**说清楚你做了什么、期望看到什么、实际看到了什么**。贴出完整的报错信息，贴出你改过的文件的关键片段。没有人有耐心从头看你的 zip 包，但他们通常愿意回答一个描述清楚的问题。

## 继续学习：两本永远翻不完的书

第一本是 **Iris Docs**（`https://iris-sodium.pages.dev`）。Uniform 列表、pass 列表、shaders.properties 语法、版本更新日志——这些文档是你光影开发的技术常数。它不是小说，它是地图。你需要时不时回头查。

第二本是 **别人的光影源码**。BSL 你已经读了。接下来试试：
- **Complementary** — 看 BSL 骨架怎么长出完全不同的美术风格。
- **Photon** — 看大胆的色彩和体积光用法。
- **Rethinking Voxels** — 看 voxel-based 渲染的特殊写法（这属于进阶到另一个流派了）。

每读一个别人的光影，你心里应该做一个练习：不看注释，先猜这个文件在干什么。然后验证。如果猜错了——恭喜，你学到了你不了解的东西。

## 发布：把你的光影交出去

你可能觉得自己的光影还不够"好看"，不敢发。或者觉得装了别人的资源包才有效果，没有通用性。

但有件事你要知道：**第一个版本不需要完美。**

发布一个光影的最低要求：
- [ ] Iris 能加载，不报 `ERROR`
- [ ] 有一个基本的 `shaders.properties` 配置界面
- [ ] 主世界白天看起来像个光影（不要求惊艳，只要求"不是原版"）
- [ ] 压缩成 zip，不要用 rar 或 7z
- [ ] 在 `shaders.properties` 里写好 `name = 你的光影名` 和 `author = 你的名字`
- [ ] 文件夹结构正确：根目录直接就是 `shaders/` 文件夹（不要外层再套一个文件夹）

发布渠道：
- **Modrinth** — 对光影作者最友好的平台，有专门的 shader 分类。
- **CurseForge** — 用户量大，但审核稍慢。两个平台都发最好。
- **GitHub** — 如果你的光影是开源的，放 GitHub 上方便别人 fork 和提 issue。BSL 的做法：仓库里直接有 `shaders/` 目录，任何人都能读。

版本命名建议用语义化版本：`v1.0.0`、`v1.1.0`。不要用 `final_v2_really_final` 这种——你的维护伙伴会哭。

## 风格：从模仿到"就是这个味道"

你刚写完的光影很容易看起来像"某知名光影的微改版"。这完全正常——你的手还在别人的肌肉记忆里。

但你应该慢慢找到自己的东西。审美方向从来不是凭空想出来的——而是你的几次"我想这样改试试"慢慢堆出来的。

也许你喜欢冷色调的晨光，于是把色温整体偏蓝。也许你喜欢强烈的 Bloom，让整个画面像回忆。也许你觉得下界不该是红色的——给你改成深紫和靛蓝。这些不是 bug，是你的视觉签名。

> 美术风格 = 连续的一千个"我觉得这样更好看"的累积。它不是设计出来的，是长出来的。

## 最后的话

第 0 章里有一句你可能已经忘了的话：

> "你的第一个光影不需要完美。它只需要运行。"

现在你回头看这句话——它是对的。你在第 1.1 节改了一行 `dot(color.rgb, vec3(1.0/3.0))`，整张画面变灰了。就这一行，你已经跨进了 GPU 编程的门槛。

从那以后，你学了法线、漫反射、延迟渲染、G-Buffer、SSAO、shadow mapping、体积云、PBR、视差贴图、维度分离。回过头看，你第一次看到方块表面出现光照方向感的时候，那种"它居然是这么算的"的恍然感，也许就是这门手艺给你最好的奖赏。

教程到这里就结束了。但你的 shader 才刚刚开始写。

希望有一天，我在 Modrinth 上点开某页，看到作者是你。

> 去吧。写你的光影。把天空染成你自己的颜色。

## 本章要点

- 你已经走过从 composite 全屏滤镜到 BSL 架构导读的完整学习路径。
- 社区是最快的 debug 和灵感来源：ShaderLABS Discord、Iris Discord、中文光影社区。
- 持续查阅 Iris Docs 和阅读其他光影的源码是你下一步自学的两大支柱。
- 光影不需要完美才能发布——Iris 能加载、主世界白天正常、有基本配置界面，就可以发第一版。
- 在 Modrinth / CurseForge / GitHub 上发布，使用语义化版本号。
- 美术风格来自不断的审美选择——它不是一天练成的，但你的每一次"不如这么改"都在塑造它。

这不是结束。这是你 shader 的第一日。
