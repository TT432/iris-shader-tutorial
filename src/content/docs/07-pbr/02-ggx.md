## 微表面是什么：把一面镜子敲碎

内心独白来一下：如果我把一面完美镜子放大一万倍，它在显微镜下是什么样的？它绝不是数学上的光滑平面——它上面全是微小的坑洼、划痕、起伏。每一小块仍然像一面小镜子（microfacet），反射很干净，但这些小镜子朝向各不相同。

微表面模型的核心思想就是这个：

> 不把整个表面当作一面大镜子，而是把它看作成千上万面朝向各异的小镜子。这些小镜子每个都完美反射，但整体效果取决于它们的朝向分布有多乱。

如果所有小镜子都朝同一个方向（光滑表面），你会看到一个锐利的高光。如果小镜子朝向乱七八糟（粗糙表面），入射光会被散射到四面八方，高光变成一团模糊的亮斑。

这是一个非常聪明的"骗术"——我们不真的画上万个三角形去模拟微观坑洼，而是用一个数学函数来描述"这些小镜子的朝向统计规律"。

---

## 三个核心函数：D、G、F

20 世纪最重要的图形学论文之一——Cook 和 Torrance 在 1981 年提出了微表面 BRDF 的通用形式：

$$
f(l, v) = \frac{D(h) \cdot G(l, v, h) \cdot F(v, h)}{4 (n \cdot l) (n \cdot v)}
$$

好，别被它吓到。我们一个一个拆：

- **D(h)** — 法线分布函数 (Normal Distribution Function)：半向量 `h` 方向上，有多少比例的小镜子正好朝向它。这是高光"长什么样"的关键。
- **G(l, v, h)** — 几何衰减函数 (Geometry Function)：有多少小镜子被旁边的凸起挡住了光线（遮蔽），或者反射光被挡住（阴影）。
- **F(v, h)** — Fresnel 项：在这个观看角度下，有多少光被反射而不是被吸收。

分母的 `4(n·l)(n·v)` 是归一化因子，它保证了 BRDF 整体不违反能量守恒——你不必深究它为什么正好是 4，知道它来自微表面几何系的坐标变换就够了。

> D 决定高光形状，G 决定边角暗不暗，F 决定斜看亮不亮。三个函数是一家人，缺一个就假。

---

## GGX：其中一个 NDF 选项

行业内常见的 D 函数有好几种：Beckmann（窄高光）、Blinn-Phong（早期游戏常用）、GGX（也叫 Trowbridge-Reitz）。MC 光影社区（包括 BSL）几乎一致选了 GGX，因为它的"长尾"(long tail) 特性特别适合模拟现实里常见的粗糙表面——那些高光边缘不会生硬地截断，而是优雅地散开。

GGX 的 NDF 写出来是这样的：

$$
D(h) = \frac{\alpha^2}{\pi \big((n \cdot h)^2(\alpha^2 - 1) + 1\big)^2}
$$

`α`（alpha）是**粗糙度**（roughness）的平方：`α = roughness × roughness`。你要注意：大部分渲染器里 roughness 取值 0 到 1，α 是它的平方——这让粗糙度滑杆的视觉效果更线性。

`n·h` 是宏观法线 `n` 和半向量 `h` 的点积。半向量是视线 `v` 和光线 `l` 之间的正中间方向：`h = normalize(v + l)`。

这里的直觉：当 `n·h` 接近 1（小镜子正对着半向量），分母趋向 `(α² - 1 + 1)² = α⁴`，D 趋向 `1/(πα²)`——这很大，表示小镜子刚好面对半向量的概率高。当 `n·h` 远离 1，分母变大，D 迅速变小。

---

## G 和 F：幕后的两位

几何遮蔽也用 GGX 配套的方案——Smith 遮蔽函数配合 Schlick-GGX 近似：

$$
k = \frac{(roughness + 1)^2}{8}
$$

$$
G_1(x) = \frac{n \cdot x}{(n \cdot x)(1 - k) + k}
$$

$$
G(l, v, h) = G_1(l) \cdot G_1(v)
$$

`G₁` 算的是"从方向 x 看过去，有多少比例不被遮挡"。光线方向算一遍（光源→表面），视线方向算一遍（表面→眼睛），两者相乘就是最终几何衰减。

Fresnel 项用 Schlick 近似就够用：

$$
F = F_0 + (1 - F_0)(1 - h \cdot v)^5
$$

`F₀` 是垂直入射时的反射率——绝缘体大约 4%，金属可以高达 80%~100%。这个值正是从 specular 贴图读取的 metalness/F0 通道（第 7.1 节提过）。

## 把公式变成代码的感觉

你可能已经在想：这么多希腊字母和分数，改成 GLSL 会是什么鬼样子？好吧，其实核心部分比你想象的短。一个简化的 GGX 调用大概长这样：

```glsl
float D_GGX(float NoH, float roughness) {
    float a = roughness * roughness;
    float a2 = a * a;
    float denom = NoH * NoH * (a2 - 1.0) + 1.0;
    return a2 / (PI * denom * denom);
}

float G_Smith(float NoV, float NoL, float roughness) {
    float k = (roughness + 1.0) * (roughness + 1.0) / 8.0;
    float G1V = NoV / (NoV * (1.0 - k) + k);
    float G1L = NoL / (NoL * (1.0 - k) + k);
    return G1V * G1L;
}

vec3 F_Schlick(float VoH, vec3 F0) {
    return F0 + (1.0 - F0) * pow(1.0 - VoH, 5.0);
}
```

`NoH`、`NoV`、`NoL`、`VoH` 分别是 `n·h`、`n·v`、`n·l`、`v·h` 的缩写。GLSL 里用这种"两个字母一个 o"的命名方式非常常见——它同时告诉你"谁和谁在点积"。

## BSL 里怎么写的

你现在不需要读懂 BSL 的 `lib/surface/ggx.glsl` 全部内容，但我希望你知道：当你准备好了，可以打开它，你会看到这些我们刚拆解过的函数——`calcGGX()` 把它们组装成完整的 Cook-Torrance 高光，处理了透光、IBL（基于图像的光照，第 11 章会讲）和多种材质分支。

BSL 的代码比我们上面的简化版多了很多细节和优化，但骨架是一模一样的：D 算高光形状，G 算遮蔽，F 算斜角反射率，三个一乘再一除，就是片元高光。

> GGX 不是"一个公式"，而是一套可以拆开替换的零件组。换一个 D 函数，高光就换张脸；换一个 F 函数，边角反光就不同。理解了每个零件的职责，你就能自己调。

## 本章要点

- 微表面模型把粗糙表面看作大量朝向不同的小镜子，用数学统计替代几何建模。
- Cook-Torrance BRDF 由三项组成：D（法线分布，决定高光形状）、G（几何遮蔽，决定遮挡暗角）、F（Fresnel，决定斜角反射率）。
- GGX 是 MC 光影社区的主流 NDF，因"长尾"特性适合表现自然粗糙表面。
- 粗糙度 `α` 取 roughness 的平方，滑杆视觉更线性；`n·h` 越大，微表面朝向半向量的概率越高。
- GLSL 实现比公式短得多——核心就三个小函数，用 `o` 中间字母的命名惯例表示点积。
- BSL 的 `lib/surface/ggx.glsl` 将这些零件组装成完整的高光管线，是你进阶学习的"标准答案"。

这里的要点是：GGX 的高光形状由 D 函数里那个 `α²` 和 `(n·h)²` 的比值决定。粗糙度大 → α 大 → 分母更大 →D 整体变小且更"胖"——高光就散开了。你不需要背诵公式，但你需要理解这个机械联动。
