这一节我们会讲解：

- 安装 Iris（光影加载器）
- 配置 VS Code + GLSL 插件
- 打开调试模式（随时看到着色器报错）
- 下载空白模板并验证一切正常

这一节没有太多理论。我们的目标是：**15 分钟之内，让你的 Minecraft 加载一个你可以修改的光影包。** 如果你已经装好了这些，可以直接跳到 0.4。

好吧，开始动手。

---

## 第一步：装 Iris

Iris 是一个 Fabric/NeoForge 模组。你需要先有一个装了模组加载器的 Minecraft。

如果你已经有一个 1.21+ 的 Fabric 或 NeoForge 客户端，跳过下面的话直接装 Iris 就行。

如果你还没有——最简单的办法是装 [Prism Launcher](https://prismlauncher.org/)。它能在创建实例时自动帮你装好 Fabric 和 Iris——勾几个框就行，不用手动折腾。

装好 Iris 后，启动 Minecraft。进入 **选项 → 视频设置 → 光影**。如果看到一个"光影包"选择界面（哪怕里面是空的），说明 Iris 装好了。

>  **重要**：确认你用的是什么版本的 Minecraft。Iris 支持 1.16.5~1.21.x。本教程假设你在用 1.21+，虽然大部分代码在较早版本也能跑。

---

## 第二步：装 VS Code 和插件

你可以用任何文本编辑器写着色器——甚至记事本都可以。但我**强烈建议**你用 VS Code。原因很简单：它能高亮 GLSL 语法，能帮你发现拼写错误，还能直接跳转 `#include` 的文件。

如果你还没装 VS Code：[下载地址](https://code.visualstudio.com/)。

装好后，打开 VS Code，在左侧扩展面板里搜索并安装这两个插件：

1. **GLSL Syntax for VS Code**（作者：GeForceLegend）
   - 给 `.vsh`、`.fsh`、`.glsl` 文件加上语法高亮
   - 安装后，你的着色器代码就不再是一片惨白

2. **Language Server for Minecraft Shaderpacks**（作者：GeForceLegend）
   - 能提示 `#include` 的路径补全
   - 能检查一些基础的语法错误
   - 能让你 Ctrl+点击跳转到被 include 的文件

两个都装好后，随便建一个文件，后缀名改成 `.fsh`，输入 `#version 330`——如果这行字变色了，就说明插件生效了。

---

## 第三步：打开调试模式（这很关键）

默认情况下，如果你的光影代码有错误，Iris 会**静默地**禁用你的光影包、换回原版渲染，只在日志里留一条毫不起眼的错误信息。对于开发者来说——这简直是一场噩梦。

所以你需要打开调试模式：

1. 进入 Minecraft **选项 → 视频设置 → 光影**
2. 选中你的光影包（在列表中高亮它）
3. 按下 **Ctrl + D**

你会看到屏幕下方出现一条提示："Debug mode enabled"。

打开调试模式后，只要你的着色器代码有任何编译错误，**错误信息会直接显示在屏幕上**——红色的文字，告诉你哪个文件的第几行出了问题。

相信我，当你深夜两点还在排查"为什么画面全黑"的时候，你会感谢这一步骤的。

另外还有一个好习惯：**让 Minecraft 启动器保持打开状态。** 在官方启动器里，去设置 → 勾选"启动 Minecraft 时保持启动器打开"和"打开输出日志"。这样你就能随时看到完整的日志。

如果你用 Prism Launcher——更好，它自带一个实时更新的日志窗口。

---

## 第四步：下载空白模板

你当然可以从零开始建一个光影包。但那样的话，你需要手动创建几十个基础文件、写一堆重复的 `#version 330`、处理各种边界情况……

或者——你可以直接用社区维护的空白模板。省去一小时的重复劳动。

我们用的是 [ShaderLABS 的 Base-330 模板](https://github.com/shaderLABS/Base-330)。这是 MC 光影社区的事实标准——Iris 官方教程用的也是它。

1. 打开 [Base-330 的 GitHub 页面](https://github.com/shaderLABS/Base-330)
2. 点击绿色的 **Code** → **Download ZIP**
3. 解压到 `.minecraft/shaderpacks/` 目录
4. 把文件夹重命名为 `Base-330`（或者你喜欢的任何名字）

目录结构应该长这样：

```
.minecraft/
 shaderpacks/
     Base-330/
         shaders/
            composite.fsh
            composite.vsh
            gbuffers_terrain.fsh
            gbuffers_terrain.vsh
            ... (还有很多)
            shaders.properties
         LICENSE
         readme.md
```

如果你看到的不是这样——比如所有 `.fsh` 和 `.vsh` 文件没有在 `shaders/` 文件夹里——那就是解压时多包了一层文件夹。把里面的内容提出来就行。

---

## 第五步：确认一切正常

现在启动 Minecraft（关闭之前别忘了按 Ctrl+D 打开调试模式），进入**选项 → 视频设置 → 光影**。你应该在列表里看到 `Base-330`。

选中它，点"应用"。回到游戏。

如果画面和原版 Minecraft 一模一样——**恭喜！一切正常。** Base-330 是一个"什么都不改"的模板，它的作用就是确保管线能跑通。没有任何视觉效果是正常的。

如果画面全黑、或者出现红字报错——检查：
1. 你的 Minecraft 版本是否  1.17？
2. Iris 是否正常加载？（按 F3，左上角应该能看到 Iris 的版本号）
3. `shaders/` 文件夹是否在正确的位置？

---

## 一个快速的测试：让屏幕变色

不等下一章了，我们现在就做一个小小的测试来确认**你真的能修改着色器并看到效果。**

用 VS Code 打开 `shaders/composite.fsh`。找到这行：

```glsl
color = texture(colortex0, texcoord);
```

把它改成：

```glsl
color = texture(colortex0, texcoord) * vec4(1.0, 0.5, 0.5, 1.0);
```

保存。切回 Minecraft，按 **F3 + R**（重载光影）。

你的世界应该变得**偏粉红色**。

如果看到了——你刚才成功修改了一个 GPU 上的着色器程序。200 万个像素，每一个都被乘以 `vec4(1.0, 0.5, 0.5, 1.0)`——红色保持 100%，绿色减半，蓝色减半。整体偏红。

如果没有——别慌。回到第四步，确认模板文件在正确的位置。也检查一下 `composite.fsh` 的第一行是不是 `#version 330 compatibility`。

---

## 本章要点

你已经完成了所有准备工作：

1.  Iris 安装——你的 Minecraft 能加载光影了
2.  VS Code + 两个 GLSL 插件——你的代码会有语法高亮
3.  调试模式（Ctrl+D）——错误信息会直接显示在屏幕上
4.  Base-330 模板——你有一个可以随时修改的空白光影
5.  F3+R 快速重载——改代码、按快捷键、看效果

如果粉红色的测试通过了，那么**你已经具备了写着色器的所有基础设施**。从现在开始，每一章你都可以一边看、一边改代码、一边看效果。

> **这里的要点是：别在环境上折腾太久。15 分钟搞定基础，然后立刻开始写代码。工具永远是为内容服务的。**
